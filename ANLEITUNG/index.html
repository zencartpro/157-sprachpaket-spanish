Dieses Sprachpaket stellt eine komplette spanische Übersetzung (Übersetzunggrad 100%) für das Frontend der Grundversion bereit. Es ist nur für 1.5.7h deutsch geeignet!
Die Sprachdefinitionen sind vollständig für die deutsche Zen Cart Version 1.5.7h angepasst, so dass es auch mit PHP 8.2 keine Probleme geben sollte.
Dieses Sprachpaket geht davon aus, dass im Shop CSS Buttons verwendet werden und liefert keine spanischen Grafikbuttons mit.

Da es seit Zen Cart 1.5.7 deutsch nicht mehr nötig ist, die Administration auf andere Sprachen umzuschalten (außer zum Bearbeiten von Define Pages unter Tools > Seiteneditor) stellt dieses Sprachpaket keine spanische Übersetzung für den Adminbereich bereit.
Im Adminbereich gibt es nur wenige Dateien, die spanische Texte für Kunden enthalten müssen, da sie für die Bestellstatusaktualisierung, Emailversand, usw. an Kunden herangezogen werden, die auf spanisch bestellt haben:

DEINADMIN/includes/languages/spanish/extra_definitions/orders_status_updates_admin.php
DEINADMIN/includes/languages/spanish/email_extras.php
DEINADMIN/includes/languages/spanish/orders.php

Die entsprechenden Definitionen dieser Dateien sind auf spanisch übersetzt, alle andere Definitionen des Adminbereichs sind auf deutsch.

Falls Sie einen komplett spanischen Adminbereich benötigen, z.B. weil Ihr Shop von einer Person administriert wird, die ausschließlich spanisch spricht, dann müssen Sie die entsprechenden Admin Definitionen selbst übersetzen.

Installation

Voraussetzungen


Dieses Sprachpaket ist nur für die deutsche Zen-Cart 1.5.7h Version geeignet
Stellen Sie sicher, dass Sie Ihren Shop auf 1.5.7h aktualisiert haben, bevor Sie diese Sprachpaket installieren.
Die Dateien sind auf dem Stand von 1.5.7h!

Vor dem Einbau dieser Änderungen:
BACKUP von Shop und Datenbank machen!
Keine Haftung, Verwendung auf eigene Gefahr!
BACKUP gemacht? Ok, dann weiterlesen...

Die Installation erfolgt in folgenden Schritten. Halten Sie diesen Ablauf ein!

1)
Im Ordner NEUE DATEIEN dieses Downloads den Ordner DEINADMIN auf den Namen Ihres Adminverzeichnisses umbenennen, anschließend mit den Ordnern email und includes in der vorgegebenen Struktur ins Zen-Cart Verzeichnis hochladen, dabei werden keinerlei bestehenden Dateien überschrieben.

2)
In der Zen-Cart Administration unter Lokalisation > Sprachen auf den Button "Neue Sprache"

Wie folgt ausfüllen

Ländercode: 34
Name: Español
Code: es
Verzeichnis: spanish
Sortierung: 8

Auf den Button "Einfügen"

3)
Im Ordner SQL dieses Downloads die spanish.sql mit einem Texteditor öffnen und den Inhalt kopieren.
Inhalt in der Zen Cart Administration unter Tools > SQL Patches installieren ins Eingabefeld einfügen und absenden.
Dadurch werden die Ländernamen für die spanische Sprachversion ergänzt.
Dieser Schritt ist unbedingt erforderlich, damit in der spanischen Version Ländernamen zur Verfügung stehen!

Hinweis:
Beim Hinzufügen der Sprache unter Lokalisation > Sprachen werden die Namen und Beschreibungen der Artikel und Kategorien für die spanische Sprache mit den bestehenden Werten der Sprache vorbefüllt, auf die die Shopadministration geschaltet ist.
Falls Sie im Shop auch die englische Sprache aktiv haben und die spanischen Artikel lieber mit englischen statt deutschen Werten vorbefüllen wollen, dann schalten Sie die Administration auf englisch um, bevor Sie die Sprache hinzufügen!

Nach dem Hinzufügen der Sprache überarbeiten Sie nun die Kategorie/Artikelnamen/Artikelbeschreibungen für spanisch.
Falls Sie EZ Pages verwenden, übersetzen Sie nun auch die jeweiligen Inhalte unter Tools > EZ Pages.
Da hier die spanischen Seitentitel noch nicht befüllt sind, kann es unter PHP 8.2 zu einigen Warning Logs kommen bei der Überarbeitung. Diese sind völlig harmlos und können gelöscht werden. Sie werden nicht mehr auftreten nachdem die spanischen Titel ergänzt sind.
Und passen Sie unter Lokalisation > Bestellstatus Ihre Bezeichnungen für die spanische Version an.
